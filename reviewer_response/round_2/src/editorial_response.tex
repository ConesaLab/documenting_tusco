\subsection*{Request 1: Data Availability --- Reporting Summary vs.\ Manuscript}

\begin{editorrequest}
The ``Data'' section from the reporting summary says: ``Publicly available LRS datasets: LRGASP Consortium (Synapse, syn25007472); RNA degradation (ENA, PRJEB53210); post-mortem brain (EGA, EGAS00001006542). In-house mouse kidney/brain data: ENA (PRJEB94912, secondary ERP177675). TUSCO gene sets: https://github.com/ConesaLab/SQANTI3 and https://tusco.uv.es. All scripts: https://github.com/ConesaLab/documenting\_tusco. Source data are provided with this paper.'' The Data availability section from the main manuscript text is different. Please ensure that the wording in these two sections is exactly the same.
\end{editorrequest}

\begin{responseboxnew}
We have regenerated the Reporting Summary so that its Data section now contains the same text as the Data Availability section in the main manuscript. Both sections are now identical.
\end{responseboxnew}

\begin{editorrequest}

\begin{responseboxnew}
We have renumbered the affiliations so that they appear in strictly sequential order upon first mention. The previous numbering assigned [5] (L.~Ferr\'{a}ndez-Peral's current address) before [4] (A.~Frankish's affiliation). We swapped the numbering of these two affiliations:

\begin{itemize}[nosep]
\item \textbf{[4]} Current address: Biozentrum, University of Basel, Basel CH-4056, Switzerland
\item \textbf{[5]} European Molecular Biology Laboratory, European Bioinformatics Institute, Wellcome Genome Campus, Hinxton, Cambridge CB10\,1SA, United Kingdom
\end{itemize}

The affiliation numbers now appear in order 1, 2, 3, 4, 5 following the author sequence.
\end{responseboxnew}
